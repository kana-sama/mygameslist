# Моя игровая библиотека

Статичный сайт для личной базы игр: тирлист, каталог, отзывы, заметки,
коллекции и локальные правки без сервера. Опубликованная база хранится в
`public/data/library.json`, а незакоммиченные изменения — только в
`localStorage` текущего браузера.

## Разработка

Требуется Node.js 22.13 или новее.

```sh
npm ci
npm run dev
```

Проверки:

```sh
npm test
npm run build
```

## Публикация данных

1. Откройте окно «Локальные правки» на сайте.
2. Проверьте diff и скопируйте команду.
3. Вставьте её в терминал внутри локального клона репозитория.
4. Скрипт проверит базу, создаст локальный commit и запустит production preview.
5. После проверки остановите preview и самостоятельно выполните
   `git push origin main`.
6. Когда GitHub Pages обновится, перезагрузите сайт: уже опубликованные
   локальные операции исчезнут автоматически.

Команда никогда не выполняет `git push` и не затрагивает файлы кроме
`public/data/library.json`.

## GitHub Pages

Workflow находится в `.github/workflows/deploy.yml`. После добавления remote
выберите в настройках репозитория **Settings → Pages → Source → GitHub Actions**.
Каждый ручной push в `main` проверит проект, соберёт `dist` и опубликует его.

## Safari и резервные копии

Локальный патч не синхронизируется между устройствами и может быть удалён
браузером. Окно diff показывает консервативный Safari-бюджет в 4 MiB и позволяет
в любой момент скачать patch-файл. Встроенные картинки автоматически
уменьшаются и конвертируются в WebP перед сохранением.
